How it works

Buyo and Sella complete a paid API call using x402, and the payment is approved with Ledger clear signing. Here is each step in plain language.

  1. Meet. Buyo needs a forecast. Sella offers her Weather Oracle, priced over x402.
  2. 402 Payment Required. Buyo's first request returns a real HTTP 402 with payment terms: pay 0.01 USDC to Sella on Base, with a nonce and an expiry. The 402 status code is how x402 asks a client to pay before continuing.
  3. Authorize. Buyo builds an EIP-3009 transferWithAuthorization message. That is EIP-712 typed data: a structured instruction to move 0.01 USDC to Sella, without broadcasting a separate transaction first.
  4. Clear signing on Ledger. The authorization goes to the Ledger signer. The plan is to drive a Ledger device with the Ledger Device Management Kit and the Ledger Ethereum Signer Kit, so the device shows exactly what is being signed and a press and hold approves it, with the private key staying in hardware. In this build the signature is simulated, so it runs anywhere; the real signing on the Speculos emulator arrives in a later phase.
  5. Settle on Base. The signed authorization goes to a facilitator that submits to Base. In this demo, settlement is simulated.
  6. Deliver. With payment settled, Sella returns the forecast. The paid request is complete.

x402 makes agent payments fast and open, and it pairs nicely with Coinbase's onchain tooling. This demo builds on x402 and adds one option: keep the signing key in hardware.

About

Agent Pays Agent is a small demo of the Ledger Agent Stack idea: an AI agent can propose a payment, but a Ledger device has the final say and signs it, so the signing key stays in hardware.

Buyo (the buyer agent) and Sella (the seller agent) make a buy and sell paid over a real x402 flow. When it is time to pay, the agent builds an EIP-712 transferWithAuthorization (USDC on Base) and hands it to the Ledger signer for clear signing, where you see exactly what you approve.

It is designed around Ledger's official tools, the Ledger Device Management Kit and the Ledger Ethereum Signer Kit, which sign on a Ledger device run through the Speculos emulator. In this build that signature is simulated so the demo runs anywhere; the real Speculos signing arrives in a later phase.

x402 makes agent payments fast and open, and it pairs nicely with Coinbase's onchain tooling. This demo builds on x402; it is not a replacement for it.

How to play

  1. Watch Buyo and Sella walk in and meet.
  2. Read the dialogue and tap a choice to advance.
  3. At payment time, check the Ledger signer card. It shows exactly what you sign.
  4. Press and hold "Hold to sign" to approve, or tap "Reject" to cancel.
