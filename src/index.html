What just happened?

Two AI agents just completed a paid API call using x402, an open payment standard. Here is each step in plain language.

  1. Request. Agent A asked Agent B for the Weather Oracle by calling GET /weather. No payment had been sent yet.
  2. 402 Payment Required. Agent B answered with HTTP status 402 and a price of 0.01 USDC on Base. The 402 status code is how x402 tells a client that payment is needed to continue.
  3. Build the authorization. Agent A prepared an EIP-3009 transferWithAuthorization message. This is EIP-712 typed data: a structured instruction that says move 0.01 USDC to Agent B, without broadcasting a separate transaction first.
  4. Sign on hardware. The authorization was handed to a Ledger device. The screen showed the exact amount and recipient, and a physical button press approved it. The private key stayed inside the device the whole time. This is the one option this demo adds on top of x402.
  5. Settle on Base. The signed authorization went to a facilitator, a small service that submits the payment to the Base network. Base confirmed it in about two seconds.
  6. Deliver. With payment settled, Agent B returned the weather data to Agent A. The paid request was complete.
