What just happened?

Buyo and Sella just completed a paid API call using x402, an open payment standard. Here is each step in plain language.

  1. Meet. Buyo needs a forecast. Sella offers her Weather Oracle, priced over x402.
  2. 402 Payment Required. Buyo's first request returned a real HTTP 402 with payment terms: pay 0.01 USDC to Sella on Base, with a nonce and an expiry. The 402 status code is how x402 asks a client to pay before continuing.
  3. Authorize. Buyo built an EIP-3009 transferWithAuthorization message. That is EIP-712 typed data: a structured instruction to move 0.01 USDC to Sella, without broadcasting a separate transaction first.
  4. Sign on hardware. The message went to a Ledger Nano. The screen showed the amount and recipient in plain text, and a button press approved it. The private key stayed on the device the whole time. That is the one option this demo adds on top of x402.
  5. Settle on Base. The signed authorization went to a facilitator that submits to Base. In this demo, settlement is simulated.
  6. Deliver. With payment settled, Sella returned the forecast. The paid request is complete.

x402 makes agent payments fast and open, and it pairs nicely with Coinbase's onchain tooling. This demo simply adds one option: keep the signing key in hardware.

The addresses, nonce, signature, and transaction hash are realistic-looking placeholders. The EIP-712 message is constructed for real; the signature is simulated for now and settlement is simulated.
